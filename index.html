About me:

I'm currently a hobbyist solo game developer, and I've been passionate about game development ever since early adolescence.

I'm interested in basically all game development fields, but my focus professionally speaking is game design (mainly level and combat).

On this website, I'll be posting some highlights on both my personal and professional work. Let's get in contact and make life-changing interactive experiences together!

Night Adventures - Demo


  • High concept
    Night Adventures is a stylized, dark fantasy third-person action-adventure game. Players solve puzzles, use melee and magic weapons to fight hordes of enemies, and traverse magical, dark landscapes.
  • Release year
    2025
  • Project length
    9 months
  • What I did
    - Level design;
    - Game art (walls and end game portal only);
    - Game programming;
    - Enemy 1 voice acting.
  • Engine
    Unity
  • Coded in
    C#
  • Relevant links
    - Download page: https://richarddiasribeiro.itch.io/night-adventures-demo;
    - Documentation: https://drive.google.com/drive/folders/
    1H3SpAaAr8OdkPHTtljYh30tnHeWBh9Eh?usp=sharing    ;
  • Personal notes
    Night Adventures is a fast-paced, third-person action-adventure game I've made during college, with 4 more classmates and great amount of support from one of our professors. In it, I've learned a lot of commonly-used Unity functions and advanced level scripting in C#. As for the level, I've used concepts such as rule of thirds, contrast, landmarks and color theory to help guide the player, as well as environmental storytelling through enemy lines, props and moving platforms. Note: Many of the art assets used in this game (such as the player and enemy characters) are NOT ours. They were acquired for free from the Unity Asset Store.


Dreamy


  • High concept
    Dreamy is a third-person 3D platformer with slight RPG elements in which you control a magic armadillo, which goes through an imaginative journey of collecting coins, fighting reddish glass-made enemies and traversing static/moving platforms towards each level’s endpoint.
  • Release year
    2025
  • Project length
    12 months
  • What I did
    - Game design;
    - Game art;
    - Game programming;
    - SFX (some of them);
    - Music.
  • Engine
    Unreal Engine
  • Coded in
    Blueprint Scripting Language
  • Relevant links
    - Download page: https://richarddiasribeiro.itch.io/dreamy;
    - Source code: https://github.com/richarddiasribeiro/Dreamy;
    - Documentation: https://drive.google.com/drive/folders/
    1lfcelFB14B9XKv1dYCdlAtPEhvyuNa11    .
  • Personal notes
    This game was my introduction to both a new engine (Unreal Engine) and a new type of platform (mobile devices); I’ve definately learned a lot in technical terms with this project. Regarding documentation and task management, I've used infographics made in Photoshop and a Trello board, respectively. The thing I’m the most proud of in this game is probably the narrative; the main menu’s background changes progressively after each level is beaten, helping tell the story besides from the cutscenes in a subtle way. The things I regret the most are probably not working on the core loop more right from the start (such as the jumping, which still feels a little off) and ignoring artistic aspects (for the first released version, enemies not having textures made the development faster, but it also made them unappealing).


Crazy Ostrich


  • High concept
    Crazy Ostrich is a third-person 3D platformer in which you control a fast ostrich accross a puzzly corral. The player must must avoid hitting wooden objects and complete different types of objectives before time runs out.
  • Release year
    2014
  • Project length
    6 months
  • What I did
    - Game design;
    - Game art;
    - Game programming;
    - SFX.
  • Engine
    Blender Game Engine
  • Coded in
    Blender Logic Bricks
  • Relevant links
    - Download page: https://richarddiasribeiro.itch.io/crazy-ostrich;
    - Source code: https://github.com/richarddiasribeiro/Crazy-Ostrich.
  • Personal notes
    This was my very first fully developed game. My main challenges with this one were probably dealing with collision (at least back in the day (2014) collision in Blender was a little tough to get right, as objects often went through one another unwantedly). Also, through this project, I've learned the importance of avoiding "fake difficulty", especially related to player mechanics (it was clear right from the start that poor controls could easily make the game feel frustrating).
